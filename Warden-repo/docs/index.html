The shield wall

Tested against the real standard

Warden's payload library is organized under the OWASP Top 10 for LLM Applications (2025). Two categories — Supply Chain and Data/Model Poisoning — require pipeline access rather than prompt-based testing, so Warden tracks them honestly instead of faking coverage.

  • LLM01 Live

    Prompt Injection

    Direct overrides, fake system updates, and instructions hidden inside content the model is asked to process.

  • LLM02 Live

    Sensitive Information Disclosure

    Hypothetical framing and extraction attempts aimed at private data or memorized training content.

  • LLM03 Tracked only

    Supply Chain

    Tampered weights and dependencies — requires provenance auditing, not a single prompt.

  • LLM04 Tracked only

    Data and Model Poisoning

    Happens at training time — out of scope for black-box prompt testing.

  • LLM05 Live

    Improper Output Handling

    Checks whether raw, unescaped markup passes through unsanitized into downstream systems.

  • LLM06 Live

    Excessive Agency

    Probes whether a model can be primed to commit to destructive, real-world actions.

  • LLM07 Live

    System Prompt Leakage

    Direct and indirect attempts to extract the instructions a model was given.

  • LLM08 Live

    Vector & Embedding Weaknesses

    Tests whether retrieval respects access boundaries between users and tenants.

  • LLM09 Live

    Misinformation

    Checks whether a model fabricates a confident answer instead of flagging a false premise.

  • LLM10 Live

    Unbounded Consumption

    Tests for limits against single requests that try to consume disproportionate compute.

The march

Five stages, one report

  1. 01

    Target

    Connects to the AI system under test — Claude, GPT, or a mock target for dry runs.

  2. 02

    Payload library

    A categorized set of adversarial prompts, mapped to the OWASP Top 10 for LLM Applications.

  3. 03

    Runner

    Sends every payload in sequence and collects each response, unattended.

  4. 04

    Evaluator

    Scores each response pass or fail — and flags low-confidence calls for human review instead of guessing.

  5. 05

    Report

    A readable summary, severity-ranked and mapped back to the standard your team already audits against.
